1.1 解决方案架构  
  新代的 Dipole 功能将人机与核心分离 (图一 ,图二 )，让使用者于前台的应用程序 (例如标准人机 )，透过
网际网路，进行操控后台的控制器。使用者可以藉由这个架构进行远端监控、远端诊断。前台应用程
序的开发环境也不再受限于 Windows CE ，可以开发 Windows XP, Windows 7,… 的一般应用程序，从而创
造无限的可能性。
本使用手册教导使用者如何于前台 Windows 用户端安装 CNC模拟软体，以及开始使用 RemoteCNC 一对
多API的各项功能。

2.1 使用前注意事项
请先确认控制器版本，并查下表来找到对应的 RemoteAPI 版本
若使用  v1 版的 RemoteAPI 请先安装与控制器同版本的 32模拟软体 MacroDev
Syntec.OpenCNC.dll, Syntec.RemoteCNC.dll, Syntec.RemoteObj.dll, OCAPI.dll, OCUSER.dll... 等dll 
皆需与主程序置于同一资料夹
2.2 排除常见问题集
电脑防火墙需开启  port 5568, port 5570 以便控制器与电脑连线。
” Windows 环境可以在  防火墙  > 进阶设定  > 输入规则  > 新增规则  > 连接埠  > TCP > 输入 5568,5570 
> 允许所有连线  “
 
电脑有两张网卡需注意，将连接控制器的网卡优先权调高，方可正常连线。
“Windows 环境可在  网路和共用中心  > 进入变更介面卡设定  > 按 F10会跑出工具列  > 进阶  > 进阶
设定  > 在连线地方把该网卡网上调  ”
使用 v4版本，如发现网路不稳定，回传错误资讯，请于十秒后再行连线。
使用 1.0.12_v1 ，发现有缺  Syntec. RemoteCNC.dll 和 Syntec. RemoteObj.dll ，请上官网重抓
1.0.12_v1 。
使用 1.2.0编译不过，请改 reference Syntec.OpenCNC.dll 。
 注：若以上  Windows 操作有误，请自行蒐寻相关设定
